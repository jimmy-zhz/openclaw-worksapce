## Security Notes

### Sensitive Information:
- **Personal documents** contain sensitive information (passport numbers, etc.)
- **Financial documents** may contain account information
- **Immigration documents** contain application numbers and personal data

### Security Practices:
1. **Encryption:** Consider encrypting sensitive documents
2. **Access Control:** Limit access to personal directories
3. **Backup:** Maintain secure backups of critical documents
4. **Redaction:** Remove sensitive information from shared versions

## File Naming Convention

Use consistent naming for easy retrieval:
```
YYYY-MM-DD_DocumentType_Description_Version.ext
```

Examples:
- `2025-04-15_StudyPermit_ApprovalLetter_v1.pdf`
- `2026-01-13_Transcript_FirstTerm_v1.pdf`
- `2026-03-31_Budget_Q1-2026_v2.xlsx`

## Version Control

For documents that change over time:
- Append version number (v1, v2, etc.)
- Include date in filename
- Maintain change log in document if applicable

## Backup Strategy

1. **Local Backup:** Regular backup to external drive
2. **Cloud Backup:** Secure cloud storage with encryption
3. **Physical Copies:** Critical documents in physical safe
4. **Verification:** Periodic verification of backup integrity

## Update Frequency

- **Financial documents:** Update monthly or as expenses occur
- **Academic documents:** Update per term or as received
- **Immigration documents:** Update as applications progress
- **Personal documents:** Update as changes occur (renewals, etc.)
